# 项目提示词：临时文件/文本分享平台

## 一、项目概述

开发一个**匿名临时文件与文本分享平台**，核心理念为"无需登录，即传即取"。用户 A 上传文件或粘贴文本后，系统生成一个短标识码（取件码）；用户 B 在任意设备上输入该标识码即可下载文件或复制文本。解决跨设备、跨平台、无法登录通讯软件时的快速传输问题。

**产品定位**：像快递柜一样简单——存件、拿码、取件。

---

## 二、核心功能

### 2.1 发送端（存件）

- **文件上传**：
  - 支持所有文件类型，不限格式
  - 单文件大小上限 1GB
  - 支持拖拽上传 + 点击选择
  - 上传过程显示实时进度条（百分比 + 速度 + 剩余时间）
  - 支持同时上传多个文件（每次分享打包为一组，共享同一个取件码）

- **文本粘贴**：
  - 提供文本输入区域，支持纯文本和代码片段
  - 无字数上限（实际限制 5MB 文本，约 500 万字）
  - 支持一次分享中同时包含文件和文本

- **有效期设置**：
  - 用户可选：1 小时 / 6 小时 / 1 天 / 3 天 / 7 天
  - 默认：1 天
  - 可配置项（管理端/配置文件）：自定义有效期选项列表、全局最大有效期
  - 过期后自动删除文件和数据库记录，不可恢复，定时任务每 10 分钟执行一次清理

- **取件码生成**：
  - 格式：6 位数字或字母（如 `A3K9X2`），方便口头传达和手动输入
  - 全局唯一，有效期内不重复
  - 过期释放的码可被后续分享复用
  - 生成后显示取件码 + 二维码 + 直接链接（三种取件方式）
  - 支持一键复制取件码

- **可选功能**：
  - 自定义取件码（如用户指定 `myfile`，若未被占用则生效）
  - 设置取件次数限制（如仅允许下载 1 次 / 5 次 / 不限次）
  - 设置取件密码（可选，留空则无密码）

### 2.2 接收端（取件）

- 输入取件码，点击查询
- 显示分享内容概要：
  - 文件：文件名、大小、类型图标、上传时间、剩余有效时间
  - 文本：直接展示文本内容（带一键复制按钮）
- 文件支持：单文件直接下载、多文件可逐个下载或一键打包下载（zip）
- 若取件码无效/已过期，友好提示"取件码不存在或已过期"
- 若设有取件密码，先弹出密码输入框验证

### 2.3 无需登录

- 所有功能对匿名用户开放
- 不收集任何用户个人信息
- 通过取件码作为唯一凭证

## 三、UI/UX 设计规范

### 3.1 设计风格

- **主题色**：酷黑（`#0A0A0A`）+ 纯白（`#FFFFFF`）+ 中性灰过渡
- **强调色**：科技蓝（`#3B82F6`）用于按钮、链接、进度条等交互元素
- **风格关键词**：极简、现代、克制、无干扰
- **字体**：中文使用思源黑体/系统默认无衬线字体，英文/数字使用等宽字体（取件码展示）
- **圆角**：统一 8-12px，按钮和卡片圆润但不过度
- **阴影**：极淡投影，营造微妙层次感
- **动效**：轻量过渡动画（200-300ms），上传/下载状态有明确视觉反馈

### 3.2 首页布局

首页即功能页，不需要单独的 landing page。用户打开即可操作。

```
┌─────────────────────────────────────────┐
│              LOGO + 一句话介绍            │
│         "无需登录，即传即取"              │
│                                         │
│   ┌─────────────┐   ┌─────────────┐     │
│   │   📤 发送    │   │   📥 取件    │     │
│   │  上传或粘贴  │   │  输入取件码  │     │
│   └─────────────┘   └─────────────┘     │
│                                         │
│         （以下为选中功能的展开区域）        │
│                                         │
└─────────────────────────────────────────┘
```

- 首屏两个大按钮/标签切换：**发送** 和 **取件**
- 发送页：拖拽区 + 文本输入区 + 有效期选择 + 提交按钮
- 取件页：取件码输入框 + 查询按钮
- 页面底部：极简 footer（项目信息、隐私说明）

### 3.3 交互细节

- **拖拽上传区**：虚线边框，拖入时高亮变色
- **取件码展示**：大号等宽字体居中显示，字符间有间距，像验证码一样醒目
- **复制反馈**：点击复制后按钮文字变为"已复制 ✓"，1.5 秒后恢复
- **加载状态**：骨架屏或 spinner，不留空白
- **错误提示**：内联提示，不使用弹窗打断用户
- **移动端适配**：完全响应式，移动端上传支持选择文件/拍照

### 3.4 暗色/亮色模式

- 默认暗色模式（酷黑背景 + 白色文字）
- 右上角可切换亮色模式（白色背景 + 深色文字）
- 跟随系统偏好自动切换

## 四、技术架构

### 4.1 推荐技术栈

| 层级 | 推荐方案 | 备选 |
|------|---------|------|
| 前端 | React/Next.js + TailwindCSS | Vue3 + TailwindCSS |
| 后端 | Python FastAPI 或 Node.js Express | Go Gin |
| 数据库 | SQLite（轻量）或 PostgreSQL | MySQL |
| 文件存储 | 本地磁盘（可配置路径） | MinIO / S3 兼容存储 |
| 缓存 | 无（SQLite 足够）或 Redis | - |
| 部署 | Docker Compose 一键部署 | 裸机部署 |

### 4.2 核心设计原则

- **轻量优先**：尽量减少外部依赖，SQLite + 本地存储即可运行
- **单机可部署**：一台 VPS 即可承载，不强制依赖云服务
- **配置文件驱动**：所有可调参数通过配置文件或环境变量控制

### 4.3 文件处理

- 上传采用分片上传（chunked upload），支持断点续传
- 大文件上传使用流式处理，不将整个文件加载到内存
- 文件存储路径结构：`/data/files/{取件码}/{原始文件名}`
- 过期清理：**后台定时任务每 10 分钟扫描，自动删除过期的文件和数据库记录**，文件从磁盘物理删除，不可恢复
- 文件下载使用流式响应（streaming response），支持断点续传

### 4.4 安全考虑

- 上传文件做病毒/恶意文件基础检测（可选，ClamAV 集成）
- 限制单 IP 上传频率（如每小时 20 次）防止滥用
- 取件码防爆破：连续 5 次错误输入后，该 IP 锁定 10 分钟
- 文件存储路径不可被直接 URL 访问，必须通过取件码接口下载
- HTTPS 强制（生产环境）
- 不记录用户 IP 到数据库（隐私保护），仅在限流时临时缓存

### 4.5 API 设计

```
POST   /api/share          # 创建分享（上传文件 + 文本 + 设置）
GET    /api/share/{code}   # 查询分享信息（不含文件内容）
POST   /api/share/{code}/verify  # 验证取件密码（如果有）
GET    /api/share/{code}/download/{fileId}  # 下载单个文件
GET    /api/share/{code}/download/all       # 打包下载所有文件
DELETE /api/share/{code}   # 发送者主动删除（通过删除令牌）
GET    /api/health         # 健康检查
```

### 4.6 数据模型

```
Share（分享记录）
├── id              主键
├── code            取件码（唯一索引）
├── type            类型：file / text / mixed
├── text_content    文本内容（如果有）
├── password_hash   取件密码哈希（可选）
├── max_downloads   最大取件次数（0 = 不限）
├── download_count  已取件次数
├── expire_at       过期时间
├── delete_token    删除令牌（创建时返回给发送者）
├── created_at      创建时间
└── files[]         关联文件列表

File（文件记录）
├── id              主键
├── share_id        关联的分享 ID
├── filename        原始文件名
├── size            文件大小（字节）
├── mime_type       MIME 类型
├── storage_path    存储路径
└── created_at      创建时间
```

## 五、配置项

通过配置文件（`config.yaml` 或 `.env`）控制：

```yaml
# 服务配置
server:
  port: 8080
  host: 0.0.0.0
  base_url: https://your-domain.com

# 分享配置
share:
  code_length: 6                    # 取件码长度
  code_charset: "ABCDEFGHJKLMNPQRSTUVWXYZ23456789"  # 排除易混淆字符 0OI1
  max_file_size: 1073741824         # 单文件最大 1GB
  max_files_per_share: 10           # 每次分享最多文件数
  max_text_size: 5242880            # 文本最大 5MB
  default_expire: 86400             # 默认有效期 1 天（秒）
  expire_options:                   # 用户可选的有效期
    - 3600        # 1 小时
    - 21600       # 6 小时
    - 86400       # 1 天
    - 259200      # 3 天
    - 604800      # 7 天
  max_expire: 604800                # 最大有效期 7 天

# 存储配置
storage:
  type: local                       # local 或 s3
  local_path: ./data/files          # 本地存储路径
  cleanup_interval: 600             # 清理间隔 10 分钟（秒）

# 安全配置
security:
  rate_limit_upload: 20             # 单 IP 每小时上传次数
  rate_limit_query: 60              # 单 IP 每分钟查询次数
  max_code_attempts: 5              # 取件码最大连续错误次数
  lockout_duration: 600             # 锁定时间 10 分钟（秒）
  enable_password: true             # 是否启用取件密码功能
  enable_custom_code: true          # 是否允许自定义取件码
```

## 六、部署要求

### 6.1 Docker Compose 一键部署

```yaml
# docker-compose.yml 示例结构
services:
  app:
    image: your-app:latest
    ports:
      - "8080:8080"
    volumes:
      - ./data:/app/data          # 文件存储
      - ./config.yaml:/app/config.yaml  # 配置
    environment:
      - TZ=Asia/Shanghai
```

### 6.2 最低硬件要求

- CPU：1 核
- 内存：512MB
- 磁盘：根据存储需求（文件总量 + 数据库）
- 网络：带宽决定上传/下载速度

### 6.3 反向代理配置

需配合 Nginx 使用，注意：
- 设置 `client_max_body_size 1G`（支持大文件上传）
- 启用 WebSocket（如果用实时进度推送）
- 配置 HTTPS（Let's Encrypt）

## 八、完整功能清单（一次性交付）

以下所有功能需在一次开发中全部完成：

### 核心流程

- 文件上传（拖拽 + 点击选择）
- 多文件上传（一组分享可包含多个文件，共享同一个取件码）
- 文本粘贴（支持纯文本和代码片段）
- 文件 + 文本混合分享
- 取件码生成与展示（大号等宽字体 + 一键复制）
- 二维码取件（生成取件二维码，手机扫码直接打开取件页）
- 直接链接取件（生成可分享的 URL）
- 取件码查询 + 文件下载 / 文本复制
- 多文件支持逐个下载或一键打包下载（zip）

### 上传体验

- 上传进度条（百分比 + 速度 + 剩余时间）
- 分片上传 + 断点续传（大文件可靠传输）
- 上传完成 / 失败状态反馈

### 分享控制

- 有效期选择（1 小时 / 6 小时 / 1 天 / 3 天 / 7 天）
- **有效期过后文件自动删除，不可恢复**
- 取件密码（可选）
- 取件次数限制（可选，如仅允许下载 1 次 / 5 次 / 不限次）
- 自定义取件码（可选，用户可指定取件码）
- 发送者可通过删除令牌主动删除分享

### UI / 主题

- 暗色模式（默认）+ 亮色模式切换
- 跟随系统偏好自动切换
- 完全响应式移动端适配
- 移动端上传支持选择文件 / 拍照

### 安全与限流

- 单 IP 上传频率限制（每小时 20 次）
- 单 IP 查询频率限制（每分钟 60 次）
- 取件码防爆破（连续 5 次错误后锁定 IP 10 分钟）
- 文件存储路径不可被直接 URL 访问

### 运维与管理

- 定时任务自动清理过期文件和记录
- 管理后台页面（查看当前活跃分享数、存储使用量、清理日志）
- Docker Compose 一键部署
- 配置文件驱动所有可调参数
- S3 / MinIO 存储后端可选（配置切换）
- API 文档（Swagger / OpenAPI 自动生成）
- 多语言支持（中文 / 英文）
